JPG

JPEG / JPG

Joint Photographic Experts Group · seit 1992

JPEG ist das meistverbreitete Format für Fotos und fotorealistische Bilder. Es verwendet eine verlustbehaftete Kompression (lossy): Bilddaten werden beim Speichern dauerhaft reduziert, um kleinere Dateigrössen zu erreichen. Die Qualität lässt sich steuern (1–100 %).

Rasterformat Verlustbehaftet 16.7 Mio. Farben Keine Transparenz
```

Wie funktioniert JPEG-Kompression?

  1. Farbraum-Umwandlung: RGB → YCbCr (Helligkeit + Farbinformation)
  2. Chroma-Subsampling: Farbinformationen werden reduziert (Auge empfindlicher für Helligkeit)
  3. Diskrete Kosinustransformation (DCT): Bild in 8×8-Pixel-Blöcke aufgeteilt und in Frequenzen zerlegt
  4. Quantisierung: Hohe Frequenzen (feine Details) werden verworfen — hier entsteht der Verlust
  5. Entropiekodierung: Verbleibende Daten werden verlustfrei komprimiert (Huffman)

Qualität vs. Dateigrösse

Simuliert die Auswirkung der JPEG-Qualitätsstufe auf die Dateigrösse (Bild: 1920×1080, 24 Bit)

JPEG-Artefakte

Bei starker Kompression entstehen sichtbare Artefakte:

  • Blockbildung: Die 8×8-Pixel-Blöcke werden sichtbar
  • Farbverfälschung: Farbverläufe zeigen Stufen (Banding)
  • Unschärfe: Feine Details und Kanten verschwimmen
  • Generationsverlust: Jedes erneute Speichern verschlechtert die Qualität weiter

✓ Vorteile

  • Sehr kleine Dateigrössen
  • Überall unterstützt
  • Ideal für Fotos
  • Einstellbare Qualität

✗ Nachteile

  • Verlustbehaftet
  • Keine Transparenz
  • Schlecht für Text/Grafiken
  • Generationsverlust
```
PNG

PNG

Portable Network Graphics · seit 1996

PNG ist ein verlustfreies Rasterformat, das speziell für das Web entwickelt wurde. Es unterstützt Transparenz (Alpha-Kanal) und eignet sich ideal für Grafiken, Logos, Screenshots und Bilder mit scharfen Kanten.

Rasterformat Verlustfrei 16.7 Mio. Farben Transparenz (Alpha)
```

PNG-Varianten

TypFarbtiefeFarbenTransparenzVerwendung
PNG-88 Bit256Ja (1 Bit)Einfache Grafiken, Icons
PNG-2424 Bit16.7 Mio.NeinFotos ohne Transparenz
PNG-3232 Bit16.7 Mio.Ja (8 Bit Alpha)Grafiken mit weichen Transparenzen

Transparenz — Alpha-Kanal

PNG-32 speichert pro Pixel 4 Bytes: R, G, B + Alpha (0 = durchsichtig, 255 = deckend). Das Schachbrettmuster zeigt transparente Bereiche:

KanalBitsWertebereichBedeutung
R (Rot)80–255Rotanteil
G (Grün)80–255Grünanteil
B (Blau)80–255Blauanteil
A (Alpha)80–255Transparenz

Verlustfreie Kompression

PNG verwendet den DEFLATE-Algorithmus (Kombination aus LZ77 + Huffman-Kodierung). Zusätzlich werden vor der Kompression Filter auf jede Pixelzeile angewendet, um die Daten für die Kompression zu optimieren.

Wichtig: Verlustfrei bedeutet, dass kein einziger Pixel verändert wird — die Datei ist aber deutlich grösser als JPEG bei Fotos.

✓ Vorteile

  • Verlustfrei — perfekte Qualität
  • Alpha-Transparenz
  • Ideal für Grafiken & Screenshots
  • Kein Generationsverlust

✗ Nachteile

  • Grosse Dateien bei Fotos
  • Keine Animation (→ APNG)
  • Langsamer zu kodieren als JPEG
  • Nicht ideal für Druck
```
SVG

SVG

Scalable Vector Graphics · W3C-Standard seit 2001

SVG ist ein Vektorformat basierend auf XML. Statt Pixel beschreibt es Formen durch mathematische Anweisungen (Punkte, Linien, Kurven). Dadurch ist es unendlich skalierbar ohne Qualitätsverlust.

Vektorformat Verlustfrei Skalierbar Transparenz XML / Text
```

Raster vs. Vektor — Zoom-Vergleich

Beobachte den Unterschied beim Vergrössern:

Rasterbild (Pixel) — wird unscharf

Vektorbild (SVG) — bleibt scharf

SVG = lesbarer Code

Eine SVG-Datei ist reiner Text (XML) und kann direkt im HTML eingebettet, mit CSS gestylt und mit JavaScript animiert werden:

<svg width="200" height="200">
  <circle
    cx="100" cy="100" r="80"
    fill="#6d28d9"
  />
</svg>

Grundlegende SVG-Elemente:

ElementBeschreibungBeispiel
<circle>Kreiscx, cy, r
<rect>Rechteckx, y, width, height
<line>Liniex1, y1, x2, y2
<path>Pfad (beliebige Form)d="M 0 0 L 100 100"
<text>Textx, y, font-size
<polygon>Vieleckpoints="0,0 50,100 100,0"

✓ Vorteile

  • Unendlich skalierbar
  • Kleine Dateigrösse (einfache Grafiken)
  • Bearbeitbar mit Code
  • Animierbar (CSS/JS)
  • Perfekt für Logos & Icons

✗ Nachteile

  • Nicht geeignet für Fotos
  • Komplexe Grafiken = grosse Dateien
  • Kann Sicherheitsrisiko sein (eingebettetes JS)
  • Rendering-Unterschiede in Browsern
```
TIFF

TIFF

Tagged Image File Format · seit 1986

TIFF ist ein professionelles Format für Druck, Archivierung und Bildbearbeitung. Es speichert Bilder in höchster Qualität (verlustfrei oder unkomprimiert) und unterstützt verschiedene Farbtiefen, Ebenen und Metadaten.

Rasterformat Verlustfrei möglich Bis 48 Bit Farbe Mehrere Ebenen
```

TransparenzJa (Alpha-Kanal)
EbenenJa — mehrere Seiten/Ebenen pro Datei
MetadatenEXIF, IPTC, XMP
Typische GrösseSehr gross (unkomprimiert)

✓ Vorteile

  • Höchste Qualität
  • CMYK für professionellen Druck
  • Mehrere Ebenen/Seiten
  • Industriestandard in Fotografie

✗ Nachteile

  • Sehr grosse Dateien
  • Nicht für Web geeignet
  • Browser-Unterstützung fehlt
  • Komplexes Format
BMP

BMP

Bitmap · Microsoft/IBM · seit 1986

BMP ist ein einfaches, unkomprimiertes Rasterformat von Microsoft. Es speichert jeden Pixel direkt — ohne jegliche Kompression. Dadurch sind BMP-Dateien sehr gross, aber die Struktur ist extrem einfach.

Rasterformat Unkomprimiert Keine Kompression Veraltet

BMP-Dateistruktur

Eine BMP-Datei hat einen einfachen Aufbau:

✓ Vorteile

  • Extrem einfache Struktur
  • Kein Qualitätsverlust
  • Schnell zu lesen/schreiben

✗ Nachteile

  • Riesige Dateien
  • Keine Kompression
  • Keine Transparenz (meistens)
  • Veraltet — heute kaum noch verwendet
```
